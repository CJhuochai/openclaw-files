# 🗃️ 记忆系统管理工具 - 使用指南

**工具名称**: `记忆系统工具.sh`  
**位置**: `/root/.openclaw/workspace/记忆系统工具.sh`  
**架构**: proactive-agent 三层记忆系统  
**目的**: 管理临时工作内存/每日日志/长期记忆之间的信息流动

---

## 🎯 概述

这个工具实现了 proactive-agent 架构的核心组件之一：**三层记忆系统**。它确保关键信息能够在正确的时间流动到正确的记忆层级，解决AI助手缺乏持久化记忆的根本问题。

### 三层架构
1. 💻 **SESSION-STATE.md** - 临时工作内存 (Write-Ahead Logging)
2. 📅 **memory/YYYY-MM-DD.md** - 每日详细日志  
3. 🧠 **MEMORY.md** - 长期记忆和核心约定

---

## 🚀 快速开始

### 1. 初始化您的记忆系统
```bash
cd /root/.openclaw/workspace
./记忆系统工具.sh init
```

这个命令将：
- 确保所有必要的文件存在
- 创建今日的日志文件
- 从长期记忆加载关键配置信息

### 2. 日常使用流程

#### 开始工作之前
```bash
./记忆系统工具.sh status  # 检查系统状态
./记忆系统工具.sh load    # 从长期记忆加载配置
```

#### 工作过程中
- 所有关键信息**手动**更新到 `SESSION-STATE.md`
- 重要事件**手动**添加到 `memory/YYYY-MM-DD.md`
- 项目约定**手动**更新到 `MEMORY.md`

#### 完成重要任务后
```bash
./记忆系统工具.sh save    # 保存工作状态到每日日志
```

## 🛠️ 详细命令说明

### `init` - 初始化记忆系统
```bash
./记忆系统工具.sh init
```

**作用**: 
- 创建memory/目录（如果不存在）
- 创建今日的日志文件
- 检查长期记忆文件
- 加载关键配置信息

**何时使用**:
- 首次设置系统
- 每天早上开始工作时
- 系统出现记忆问题时

### `save` - 保存到每日日志
```bash
./记忆系统工具.sh save
```

**作用**:
- 从SESSION-STATE提取最近的关键执行记录
- 添加到今日日志的"关键事件和进展"部分
- 时间戳自动添加

**何时使用**:
- 完成重要任务/里程碑后
- 做出重要决策后  
- 遇到有学习价值的错误时

### `load` - 从长期记忆加载
```bash
./记忆系统工具.sh load
```

**作用**:
- 从MEMORY.md提取关键项目信息和约定
- 确保SESSION-STATE包含所有必要基础信息

**何时使用**:
- 开始新项目/任务时
- 怀疑信息缺失或不一致时
- 需要刷新关键配置信息时

### `status` - 显示系统状态
```bash
./记忆系统工具.sh status
```

**输出内容**:
```
=== 三层记忆系统状态 ===

1. 🧠 临时内存 (SESSION-STATE)
   ✓ 存在，文件大小: 146 行
   📅 最近更新: 2026-02-28 17:17:08

2. 📅 每日记忆 (memory/目录)
   ✓ 存在，包含 3 天的记录
   今日日志: 2026-02-28.md
   今日内容: 8 个章节

3. 🗃️ 长期记忆 (MEMORY.md)
   ✓ 存在，文件大小: 57 行
   内容: 6 个关键约定
```

### `check` - 健康检查
```bash
./记忆系统工具.sh check
```

**检查内容**:
- 所有必需文件是否存在
- 文件是否可写
- 目录结构是否正确
- 文件权限是否适当

## 📝 工作流程示例

### 完整的工作日流程

#### 早上开始工作
```bash
# 1. 检查记忆系统状态
./记忆系统工具.sh status

# 2. 加载长期约定
./记忆系统工具.sh load

# 3. 检查今日是否已创建日志
./记忆系统工具.sh init
```

#### 工作过程中
- 接收关键信息 → **手动更新 SESSION-STATE.md**
- 做出重要决策 → **立即记录在 SESSION-STATE.md**
- 遇到问题/错误 → **详细记录到 memory/YYYY-MM-DD.md**

#### 完成重要任务后
```bash
# 保存这次任务的关键执行记录
./记忆系统工具.sh save
```

#### 结束一天工作
- 回顾今日的 memory/YYYY-MM-DD.md 文件
- 提炼有价值经验到 MEMORY.md（手动）
- 清理 SESSION-STATE.md中的临时信息

## 🔧 高级配置

### 自定义文件位置
如果您想改变文件位置，编辑脚本中的配置部分：
```bash
WORKSPACE="/root/.openclaw/workspace"
SESSION_STATE="$WORKSPACE/SESSION-STATE.md"
MEMORY="$WORKSPACE/MEMORY.md"
MEMORY_DIR="$WORKSPACE/memory"
```

### 自动化集成

您可以将记忆系统工具集成到工作流程中：

#### 在任务模板中添加
```bash
#!/bin/bash
# 示例任务模板

# 加载记忆系统
cd /root/.openclaw/workspace
./记忆系统工具.sh load

# 检查系统状态  
./记忆系统工具.sh status

# 执行任务
echo "执行任务: $1"
# ... 任务代码 ...

# 保存结果
./记忆系统工具.sh save
```

#### 定时任务集成
```bash
# 每小时检查记忆系统健康
0 * * * * cd /root/.openclaw/workspace && ./记忆系统工具.sh check
```
