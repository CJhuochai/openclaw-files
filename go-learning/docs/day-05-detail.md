## 2. 核心概念详解（面向Java开发者）
**概念是什么**：包管理入门 是 Go 学习路径中的关键节点，要求你不仅会写，还要理解设计取舍。  
**为什么重要**：它通常会直接影响代码可维护性、性能或并发安全。  
**迁移思路**：先想清楚 Java 里你会怎么做，再对照 Go 的最简洁做法，避免把 Java 写法“硬搬”到 Go。

## 3. Go vs Java 深度对比
| 维度 | Go | Java | 实战建议 |
|---|---|---|---|
| 代码组织 | 倾向小而清晰的包与函数 | 倾向类与分层结构 | 先写最小可运行版本，再逐步抽象 |
| 错误处理 | 显式返回 error | 异常机制 | 在边界层统一处理错误 |
| 并发模型 | goroutine + channel | Thread + 锁/并发包 | 优先通信，必要时加锁 |

## 4. 代码示例（可运行）
### 示例1：最小可运行
```go

    fmt.Println("Day 05: 包管理入门")

```

### 示例2：工程化写法（带错误处理）
```go
package main

import (
    "errors"
    "fmt"
)

func run(input string) (string, error) {
    if input == "" {
        return "", errors.New("input 不能为空")
    }
    return "ok: " + input, nil
}

func main() {
    out, err := run("demo")
    if err != nil {
        fmt.Println("err:", err)
        return
    }
    fmt.Println(out)
}
```

## 5. 常见坑与排错
